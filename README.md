# 阿瓦隆 Online

一个给线下玩家使用的阿瓦隆网页工具。在没有实体身份牌、任务卡、投票道具时，玩家可以直接用手机完成开局、组队、投票、任务和刺杀流程。

## 项目目标

- 支持 5 到 10 人的线下阿瓦隆局
- 用网页替代主持人和实体道具
- 核心流程稳定，避免因为误操作或刷新中断整局游戏
- 手机端可直接使用

## 快速开始

### 安装依赖

```bash
npm install
```

### 启动服务

```bash
PORT=3000 node server.js
```

默认访问地址：

- 本机: [http://localhost:3000](http://localhost:3000)
- 局域网: `http://你的局域网IP:3000`

如果要给微信或外网访问，可自行配合 Nginx/云主机转发到 80 端口。

## 核心流程

1. 房主创建房间并选择人数。
2. 玩家输入邀请码加入房间。
3. 房主开始游戏，服务端分配角色并分别下发可见信息。
4. 当前队长按该轮要求选择任务成员。
5. 全员投票决定队伍是否通过。
6. 队伍通过后，仅任务成员可以提交任务结果。
7. 好人 3 次成功进入刺杀阶段，坏人 3 次失败直接获胜。
8. 刺客在刺杀阶段选择目标，若命中梅林则坏人翻盘。

### 当前规则实现

- 5 次连续组队投票失败：坏人胜利
- 7 人及以上第 4 轮：需要 2 张失败票才算任务失败
- 好人玩家不能提交 `fail`
- 非当前操作者不能越权操作
- 任务结果只公布成败和失败票数量，不公布具体是谁出的结果
- 对局进行中禁止主动退房，避免线下局被误触打断
- 刺客“验人”功能已关闭，避免破坏推理体验

## 移动端说明

- 目标使用场景是微信内点击网页链接直接游玩
- 主要操作按钮按移动端触控尺寸处理
- 弹窗支持纵向滚动，适配小屏设备
- 页面考虑了底部安全区，减少 iPhone 刘海屏/手势条遮挡

## 最近验证

- 本地 5 人流程验证通过：建房 -> 入房 -> 开局 -> 组队 -> 投票 -> 任务 -> 下一轮

## 状态管理约定

- 实时对局状态以 `server.js` 中的 `rooms` 内存 Map 为准
- SQLite 负责用户信息、排行榜和房间基础信息
- 前端只负责展示和发起操作，最终权限判断全部以服务端为准
